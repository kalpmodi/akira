# Code of Conduct

## The Short Version

Be decent. This is a security research project. We're here to learn, break things responsibly, and help each other get better at offensive security.

---

## What We Expect

- Constructive feedback on skills and techniques
- Honest attribution - if you found a bug with Akira, say so
- Responsible disclosure - don't drop live 0days in issues
- Respect that people are at different skill levels

## What We Don't Tolerate

- Sharing findings from active engagements without authorization
- Using this project or its community to coordinate illegal activity
- Harassment, discrimination, or personal attacks
- Doxxing contributors or targets

---

## Scope

This applies to GitHub Issues, Pull Requests, Discussions, and any other space associated with this repository.

---

## Reporting

If someone is violating this, open a private vulnerability report on this repo or contact the maintainer directly via GitHub.

Violations that are clear-cut (harassment, illegal activity) will result in immediate removal. Everything else gets a conversation first.

---

## The Baseline

We follow the [Contributor Covenant v2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) where this document doesn't cover a specific case.
